new idea: we alias g++, python3, etc. to wrap around the
original command that uses its flags and arguments to set
configs for related source files

for context, we only need config for bundling and submitting
and, typically, bundling happens during submitting. since it's
always good practice to run on samples before submitting, we
can kind of bet that g++, python3, etc. will be run before
submission and our config would be present and up-to-date by
the time we submit

this is kind of the same idea i had with the browser, where we
warm up the browser whenever samples are run. that way, the
browser is typically already up by the time we submit, speeding
things up a lot
